The <table> tag creates the table. The <tr> tag is used for each table row. The <th> tag is used for table headings, and the <td> tag is used for table data cells.

Question about incorrect use of <th>

Why is it wrong to use the <th> tag for all rows just because it looks bold, and when should the <th> tag actually be used?

This thinking is wrong because the <th> tag is not for styling but for meaning. It should only be used for headings of rows or columns, not for normal data. Using it incorrectly can confuse screen readers and reduce accessibility.

Captions are important because they describe what the table is about. By default, the caption appears at the top of the table.

What is the difference between a table row and a table column?

A table row is created using the <tr> tag and runs horizontally across the table. Columns are created using <td> and <th> tags inside each row and run vertically through the table.

Why are tables suitable for this data and why should they not be used for layout?

Tables are suitable for this data because student records are structured and organized into rows and columns. This makes the data easy to read and understand. However, tables should not be used for page layout because they make the code messy, reduce accessibility, and make websites harder to maintain compared to modern layout methods like CSS.

How do table headings help screen readers and accessibility tools?

The <th> tag provides semantic meaning by identifying headings in the table. Screen readers use this structure to understand relationships between data cells. This helps users with visual impairments navigate and understand the table more easily.
